# 接続

内蔵 HTTP / WebSocket サーバーの状態確認と、OBS ブラウザソース用 URL の取得を行うパネルです。

---

## 役割

- レイヤーごとの HTTP URL 表示・コピー
- OBS へのドラッグ＆ドロップでブラウザソース追加
- サーバー稼働状態・ブラウザ接続数の確認

---

## レイヤー URL

各レイヤーに次の形式の URL が割り当てられます。

```
http://127.0.0.1:{WebSocketポート}/{レイヤー番号}
```

既定ポートは **21760**（設定パネルで変更可）。

| ボタン | 動作 |
| ------ | ---- |
| コピー | URL をクリップボードにコピー |
| OBS へドラッグ | OBS ソース一覧へドロップしてブラウザソースを自動追加 |

---

## OBS へドラッグ

1. **OBS へドラッグ** ボタンを押したまま OBS のソース一覧へドロップ
2. 設定パネルの **OBS キャンバス解像度** がソースの幅・高さに使われます
3. OBS WebSocket（設定の OBS WebSocket ポート、既定 4455）経由で既存ブラウザソースの更新も試みます

!!! note "管理者権限の OBS"
    OBS を管理者権限で起動している場合、ドラッグ＆ドロップが失敗することがあります。  
    その場合は URL コピーで手動追加してください。

---

## 接続状態表示

| 表示 | 意味 |
| ---- | ---- |
| サーバー: 稼働中 / 停止 | 内蔵 HTTP サーバーの状態 |
| ブラウザソース: 接続済み / 接続待ち | OBS 等のクライアントが WebSocket 接続しているか |
| クライアント数 | 接続中クライアントの数 |

---

## OBS ブラウザソースの推奨設定

- **シーンがアクティブになったときにブラウザを再読み込み** … チェック ON  
- ページ権限 … 初期設定（OBS ステータス情報へのアクセス）のままで可

GoodLiveWidget を OBS より後から起動した場合でも、シーン切替時に再接続・再表示されます。
